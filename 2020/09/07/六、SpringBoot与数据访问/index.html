Spring Boot与数据访问

简介

  对于DAO层（Data Access Object，也称数据访问层），Spring Boot默认采用整合Spring Data的方式来进行统一的处理（无论是SQL还是NOSQL都使用Spring Data），其中添加了大量的自动配置，引入了很多XXXTemplate、XXXRepository来简化DAO层的操作。使用Spring Data只需要进行简单的配置即可。下面将从如下几个部分来进行讲解：

  • JDBC
  • MyBatis
  • JPA

当然，Spring Data远不止这些，其中包含了很多的子项目，具体可以参考：https://spring.io/projects/spring-data

Spring Boot 2.3.0中默认使用HikariDataSource作为数据源

数据源的相关配置都在DataSourceProperties里面

自动配置原理：

  1. DataSourceConfiguration：根据spring.datasource.type配置来创建数据源，默认为HikariDataSource

    可以通过spring.datasource.type指定自定义的数据源类型

  2. Spring Boot默认支持

    • org.apache.tomcat.jdbc.pool.DataSource
    • com.zaxxer.hikari.HikariDataSource
    • org.apache.commons.dbcp2.BasicDataSource

  3. 自定义数据源类型

    自定义的DataSource，就是从配置文件中获取spring.datasource.type的值，然后利用反射创建DataSource

  4. DataSourceInitializerInvoker

    DataSourceAutoConfiguration中使用@Import注解引入了DataSourceInitializationConfiguration。而在DataSourceInitializationConfiguration中又使用了@Import注解引入了DataSourceInitializerInovker

    DataSourceInitializerInover顾名思义就是DataSourceInitializer的执行器，它实现了ApplicationListener<DataSourceSchemaCreatedEvent>，也就是说当DataSourceSchemaCreatedEvent事件发生了，这个类中的onApplicationEvent方法就会执行：

    DataSourceSchemaCreatedEvent会在schema-*.sql文件被执行时或者Hibernate初始化数据库的时候调用

    DataSourceInitializer的作用：

    • 运行建表语句（schema-*.sql）
    • 运行插入数据语句（data-*.sql）

    也就是说schema-*.sql和data-*.sql文件会被执行（默认是在类路径下，也可以通过spring.datasource.schema和spring.datasource.data来指定路径（可以指定多个），将文件放在对应的目录下，在Spring Boot应用启动时就会执行这些SQL文件

整合JPA

Spring Data与JPA

  Spring Data是为了简化Spring框架应用的数据访问技术，包括非关系型数据库、Map-Reduce框架、云数据服务和关系型数据库等等。

Spring Data包含了非常多的子项目：

Spring Data的特点

  Spring Data提供统一的API来对数据访问层进行操作，这主要是通过Spring Data Commons项目来实现的。Spring Data Commons可以让开发人员在使用关系型数据库和非关系型数据库访问技术时都基于Spring提供的统一标准（包含了CRUD、排序、分页等操作）

Spring Data与JPA的关系

JPA：Java Persistence API，Java持久化层API。是JAVA提供的持久层规范，如上图所示Hibernate、Toplink、OpenJPA等都实现了JPA规范

Spring Data XXX是Spring Data对各种数据访问技术进行自动配置（简化），例如：Spring Data Redis、Spring Data MongoDB等，它们都可以使用Spring Data提供的统一API进行持久层编程，也就是说使用了Spring Data之后就可以不用关心具体使用了何种数据访问技术，统一使用Spring Data提供的API进行持久层的编程开发。

Spring Data也提供了对JPA的支持，Spring Data JPA就是Spring Data的JPA模块，对JPA进行了自动配置和简化操作。JPA 2.0可以查看JSR 317的相关文档。
