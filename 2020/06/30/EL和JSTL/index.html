EL和JSTL

EL

EL是什么，有什么作用

EL：Expression Language，表达式语言。

作用：EL表达式是为了替代JSP页面中的表达式脚本在JSP页面中输出数据

相较于JSP表达式脚本，EL表达式就显得十分简洁

EL表达式的格式

注意：EL表达式对于null值的显式十分友好，如果表达式的值为null，则会显式为空串，而不是像表达式脚本一样显示为null字符串

EL表达式搜索域数据的顺序

pageContext→request→session→application

EL表达式输出Bean的各种属性

Person.java

普通属性、数组属性、List属性、集合属性的输出

注意：

  1. EL表达式是通过域对象的name属性来访问的
  2. EL表达式的属性由Bean对象的get方法决定而非属性名

EL表达式的运算

语法：

EL表达式支持如下运算符

关系运算

关系运算符 说明 范例 结果
==或eq 等于 ${ 5 == 5 }或${ 5 eq 5 } true
!=或ne 不等于 ${ 5 != 5 }或${ 5 ne 5 } false
<或lt 小于 ${ 3 < 5 }或${ 3 lt 5 } true
>或gt 大于 ${ 2 > 10 }或${ 2 gt 10 } false
<=或le 小于等于 ${ 5 <= 12 }或${ 5 le 12 } true
>=或ge 大于等于 ${ 3 >= 5 }或${ 3 ge 5 } false

逻辑运算

逻辑运算符 说明 范例 结果
&&或and 与运算 ${ 12 == 12 && 12 < 11 }或${ 12 == 12 and 12 < 11 } false
` 或or` 或运算
!或not 取反运算 ${ !true }或${ not true } false

算术运算

算术运算符 说明 范例 结果
+ 加法 ${12 + 18 } 30
- 减法 ${ 18 - 12 } 6
* 乘法 ${ 2 * 3 } 6
/或div 除法 ${ 6 / 3 }或${ 6 div 3} 2
%或mod 取模 ${ 7 % 3 }或${ 7 mod 3 } 1

empty运算

empty运算可以判断一个数据是否为空，如果为空，则输出true，不为空输出false。

以下几种情况为空：

  1. 值为null的时候，为空
  2. 值为空串的时候，为空
  3. 值是Object类型数组，长度为零的时候
  4. list集合，元素个数为零
  5. map集合，元素个数为零

三元运算

格式：

如果表达式1的值为真，返回表达式2的值，否则返回表达式3的值

.点运算和[]中括号运算符

.点运算，可以输出Bean对象中某个属性的值

[]中括号运算，可以输出有序集合中某个元素的值。

并且[]中括号运算，还可以输出map集合中key里含有特殊字符的key的值

EL表达式的11个隐含对象

EL表达式中11个隐含对象，是EL表达式中自己定义的，可以直接使用

pageContext

  • pageContext：PageContextImpl类型对象，可以获取JSP中的九大内置对象

四大域对象相关

  • pageScope：Map<String, Object>类型对象，可以获取pageContext域中的数据
  • requestScope：Map<String, Object>类型对象，可以获取request域中的数据
  • sessionScope：Map<String, Object>类型对象，可以获取session域中的数据
  • applicationScope：Map<String, Object>类型对象，可以获取application域中的数据

请求参数相关

  • param：Map<String, String>类型对象，可以获取请求参数的值
  • paramValues：Map<String, String[]>，可以获取多个值的请求参数

请求头相关

  • header：Map<String, String>类型对象，可以获取请求头的值
  • headerValues：Map<String, String[]>类型对象，可以获取具有多个值的请求头

cookiie

  • cookie：Map<String, Cookie>类型，获取当前请求中的cookie信息

initParam：

  • initParam：Map<String, String>类型，它可以获取web.xml中配置的<context-param>中的参数

pageContext对象的使用

  1. 获取协议
  2. 服务器IP
  3. 服务器端口号
  4. 获取工程路径
  5. 获取请求方法
  6. 获取客户端IP
  7. 获取会话的id编号

使用示例

JSTL（重点）

JSTL：JSP Standard Tag Library，即JSP标准标签库。这是一个不断完善的开源JSP标签库

作用：替换JSP中的代码脚本，使得整个JSP页面变得更加简洁

JSTL由五个不同功能的标签库组成：

功能范围 URI 前缀
核心标签库（重点） http://java.sun.com/jsp/jstl/core c
格式化 http://java.sun.com/jsp/jstl/fmt fmt
函数 http://java.sun.com/jsp/jstl/functions fn
数据库(不使用) http://java.sun.com/jsp/jstl/sql sql
XML(不使用) http://java.sun.com/jsp/jstl/xml x

在JSP中标签库需要使用taglib指令引入

JSTL的使用步骤

  1. 引入JSTL的依赖

  2. 使用taglib指令引入对应的标签库（以核心标签库为例）

c:set（很少使用）

    • request：request域

    • application：ServletContext域
  • var：属性的key
  • value：属性的值

c:forEach

作用：遍历输出使用

属性：

  • begin：开始索引
  • end：结束索引
  • var：表示循环的变量，也是当前正在遍历到的数据
  • step：表示每次增长的量
  • items：表示当前遍历的数据源
  • varStatus：表示当前遍历到的数据的状态，有如下属性
    • step：步长（每次的增长量）
    • begin：开始索引
    • end：结束索引
    • count：当前是正在遍历的第几个元素
    • current：当前遍历的对象
    • first：对应isFirst()，是否为第一个元素
    • last：对应isLast()，是否为最后一个元素
    • index：索引
