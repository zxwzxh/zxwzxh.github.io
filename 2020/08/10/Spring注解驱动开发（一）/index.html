组件注册

@Configuration

作用：@Configuration在某个类上时会将这个类作为配置类注入到SpringIoC容器中

@Configuration本身也是@Component的别名，所以@Configuration修饰的类也会作为组件被注入到Spring IoC容器中

@Bean

作用：@Bean注解修饰的方法会作为bean注入到Spring的IoC容器中

注解的定义如下：

  • value、name：对应bean标签中的id属性
  • autowire：返回一个Autowire枚举类的对象，默认值为Autowire.NO
    • Autowire.NO：不自动装配
    • Autowire.BY_NAME：根据名称自动装配
    • Autowire.BY_TYPE：根据类型自动装配
  • initMethod：指定当bean创建时调用的方法
  • destroyMethod：指定当bean销毁时调用的方法

@ComponentScan

作用：根据定义的扫描路径，把符合规则的类装配到Spring容器中

注解的定义如下：

  • value和basePackages：用于指定包的路径，进行扫描
  • basePackageClasses：可以指定多个类或接口的class，扫描时会在这些指定的类和接口所属的包进行扫描
  • nameGenerator：bean的名称生成器，默认值为BeanNameGenerator.class
  • useDefaultFilters：是否开启对@Component、@Repository、@Service、@Controller的类进行检测，默认值为true
  • includeFilters：包含的过滤条件，Filter数组
  • excludeFilters：排除的过滤条件，Filter数组

@Filter

@Filter注解是一个定义在@Component注解中的注解，用于过滤扫描的内容

注解的定义：

  • type：用于指定过滤的类型，默认值为FilterType.ANNOTATION

    • ANNOTATION：注解类型

    • ASSIGNABLE_TYPE：指定的类型

    • ASPECTJ：按照Aspectj的表达式，基本上不会使用

    • REGEX：按照正则表达式

    • CUSTOM：自定义规则

      用法：实现TypeFilter接口

      添加自定义的配置：

  • value和classes：指定要过滤的类型，例如：Controller.class
  • pattern：匹配规则，Aspectj或者Regex的匹配规则

@Scope

@Scope相当于bean标签中的scope属性，默认值是singleton，即单例的

  • singleton：默认值，单例的，即每次从IoC获取时获得的是同一个对象，会在容器创建时将这个单实例添加到IoC容器中
  • prototype：多实例，每次从IoC容器中获取的是不同的对象（每次获取重新创建一个），每次获取时都会创建实例
  • request：同一个请求中获取的是同一个对象
  • session：同一个会话中获取的是同一个对象

@Lazy

singleton的时候会在IoC容器创建时加载，那如何修改成懒加载，即在获取时创建呢？

可以使用@Lazy标签实现，默认值为true，可以使得单例的bean变为懒加载模式

@Lazy(false)没有意义，和没写@Lazy的作用一样！

==ConditionContext的常用方法==

  • BeanDefinitionRegistry getRegistry()：返回condition匹配的持有bean定义的BeanDefinitionRegistry对象，也就是说可以获取和Bean定义有关的信息
  • ConfigurableListableBeanFactory getBeanFactory()：返回Bean工厂对象

  • ClassLoader getClassLoader()：返回被用于加载class字节码的类加载器对象

==AnnotatedTypeMetadata常用方法==

  • boolean isAnnotated(String annotationName)：返回当前的元素（扫描的类或者接口）是否包含指定的注解（或元注解）
  • Map<String, Object> getAnnotationAttributes(String annotationName)：以Map<String, Object>对象的形式返回当前元素的指定注解的属性
  • Map<String, Object> getAnnotationAttributes(String annotationName, boolean classValuesAsString)：以Map<String, Object>的形式返回当前元素的指定注解的属性，第二个参数表示是否将class对象转换为字符串存入Map中
  • MultiValueMap<String, Object> getAnnotationAttributes(String annotationName)：以MultiValueMap<String, Object>对象的形式返回当前元素的指定注解的属性
  • MultiValueMap<String, Object> getAnnotationAttributes(String annotationName, boolean classValuesAsString)：以MultiValueMap<String, Object>的形式返回当前元素的指定注解的属性，第二个参数表示是否将class对象转换为字符串存入Map中

@Import

给容器中注入组件的方式：

  1. 包扫描+@Component、@Service、@Controller、@Repository注解的方式

    缺点：只适用于自己编写的类

  2. @Bean，可以导入第三方包里面的组件

  3. @Import，使用@Bean每次都需要创建一个对象，有的时候只需要调用无参构造，此时可以使用@Import来快速导入一个组件

  4. 使用Spring提供的FactoryBean（工厂Bean），经常在Spring与其他框架进行整合时使用

作用：向IoC容器中快速注入组件

@Import注解的三种使用方式总结

  • class数组的方式，容器会自动注册这个组件，id默认为全类名
  • ImportSelector的方式，需要返回导入组件的全类名的字符串数组，id也是全类名，spring boot底层使用十分频繁（重点）
  • ImportBeanDefinitionRegistrar的方式，手动注册到bean，可以自定义bean的名称（id）

注意：这三种方式可以在同一个@Import注解中使用！

Bean的初始化与销毁

初始化：对象创建完成并且调用赋值完毕时，调用初始化方法

销毁：

  • 单实例：容器关闭时（默认）如果设置了懒加载则和多实例的销毁相同
  • 多实例：容器不会管理这个Bean，容器不会调用销毁方法
