Listener和Filter

Listener

监听器概述

Listener就是监听器，和Servlet、Filter组成了JavaWeb的三大组件。

Listener由三个部分组成

  • 事件源
  • 事件
  • 监听器

可以这样理解，事件源相当于小偷，事件是小偷偷东西，监听器就是经常。当小偷（事件源）偷东西了（事件）经常（监听器）就会对小偷进行抓捕（监听器中的方法）

监听器的分类：

  • 监听域对象创建与销毁的监听器
  • 监听域对象操作域属性的监听器
  • 监听HttpSession的监听器

监听器的配置

监听器在web.xml中的配置十分简单：

创建与销毁监听器

ServletContext相关

  • ServletContextListener：
    • void contextInitialized(ServletContextEvent event)：Web服务器启动时调用
    • void contextDestroyed(ServletContextEvent event)：Web服务器关闭时调用

HttpSession相关

  • HttpSessionListener：
    • void sessionCreated(HttpSessionEvent event)：会话创建时调用
    • void sessionDestroyed(HttpSessionEvent event)：会话销毁时调用

ServletRequest相关

  • ServletRequestListener
    • void requestInitialized(ServletRequestEvent event)：请求开始时调用
    • void requestDestroyed(ServletRequestEvent event)：请求结束时调用

相关Event的介绍

  • ServletContextEvent：ServletContext getServletContext()
  • HttpSessionEvent：HttpSession getSession()
  • ServletRequestEvent：
    • ServletRequest getServletRequest()
    • ServletContext getServletContext()

操作属性域的监听器

ServletContext相关

  • ServletContextAttributeListener
    • void attributeAdded(ServletContextAttributeEvent event)：向ServletContext域中添加属性时调用
    • void attributeReplaced(ServletContextAttributeEvent event)：向ServletContext域中更新属性时调用
    • void attributeRemoved(ServletContextAttributeEvent event)：向ServletContext域中移除属性时调用

HttpSession相关

  • HttpSessionAttributeListener
    • void attributeAdded(HttpSessionAttributeEvent event)：向session域中添加属性时调用
    • void attributeReplaced(HttpSessionAttributeEvent event)：向session域中更新属性时调用
    • void attributeRemoved(HttpSessionAttributeEvent event)：向session域中移除属性时调用

ServletRequest相关

  • ServletRequestAttributeListener
    • void attributeAdded(ServletRequestAttributeEvent event)：向request域中添加属性时调用
    • void attributeReplaced(ServletRequestAttributeEvent event)：向request域中更新属性时调用
    • void attributeRemoved(ServletRequestAttributeEvent event)：向request域中移除属性时调用

向关Event介绍

  • ServletContextAttributeEvent
    • String getName()：获取当前操作的属性名
    • Object getValue()：获取当前操作的属性值
    • ServletContext getServletContext()：获取ServletContext对象
  • HttpSessionAttributeEvent
    • String getName()：获取当前操作的属性名
    • Object getValue()：获取当前操作的属性值
    • HttpSession getSession()：获取当前的session对象
  • ServletRequestAttributeEvent
    • String getName()：获取当前操作的属性名
    • Object getValue()：获取当前操作的属性值
    • ServletRequest getServletRequest()：获取request
    • ServletContext getServletContext()：获取ServletContext对象

HttpSession的监听器

还有两个与HttpSession相关的特殊的监听器，这两个监听器的特点如下：

  1. 不用在web.xml中进行配置
  2. 这两个监听器不是监听Session的，而是监听Bean的。即让Bean类实现监听器接口，然后再把Bean对象添加到Session与中

这两个监听器是：

  • HttpSessionBindingListener：
    • void valueBound(HttpSessionBindingEvent event)：当监听器对象被添加到session中会调用这个方法
    • void valueUnbound(HttpSessionBindingEvent event)：当监听器对象从session中移除时会调用这个方法
  • HttpSessionActivationListener
    • void sessionWillPassivate(HttpSessionEvent event)：当对象感知被活化时调用本方法
    • void sessionDidActivate(HttpSessionEvent event)：当对象感知被钝化时调用被方法

Filter

什么是Filter

  1. Filter是JavaWeb的三大组件之一
  2. Filter是Java EE规范
  3. Filter的作用是：拦截请求，过滤响应

拦截请求常见的应用场景：

  1. 权限检查
  2. 日志操作
  3. 事务管理
  4. …

Filter的使用步骤

  1. 编写一个类实现Filter接口
  2. 实现doFilter方法
  3. 在web.xml中配置Filter（类似于Servlet的配置）

用户权限校验

要求：在你的web工程下，有一个admin目录。这个admin目录下的所有资源（html页面、jpg图片、jsp文件等等）都必须是用户登录之后才允许访问的。

思考：根据之前我们学过的内容。我们知道，用户登录之后都会把用户信息保存到Session域中。所以要检查用户是否登录，可以判断session中是否包含有用户登录的信息即可！！！

Filter的生命周期

Filter接口中有如下三个方法：

  • void init(FilterConfig filterConfig)：初始化方法，在web工程启动时执行
  • void doFilter(ServletRequest request, ServletResponse response, FilterChain chain)：每次拦截到请求都会执行
  • void destroy()：销毁方法，在web工程关闭时执行

FilterConfig类

FilterConfig类就是Filter的配置类。

Tomcat在创建Filter的同时也会创建一个FilterConfig类对象，其中包含了Filter配置文件的相关信息

FilterConfig的作用

  1. 获取Filter的名称，即filter-name标签中的内容
  2. 获取Filter配置中init-param中的初始化参数
  3. 获取ServletContext对象

FilterChain过滤器链

FilterChain就是过滤链，因为Filter向链条一样一次执行，只有前面的过滤器执行了FilterChain对象的doFilter方法放行之后，后面的Filter或者Servlet等Web资源才能够执行！如果Filter没有放行，那么就会被拦截，后面的该过滤器的后置代码以及链后面的内容都不会执行！

FilterChain.doFilter()方法的作用：

  1. 执行下一个Filter过滤器（如果有）
  2. 执行目标资源（没有Filter)

在多个Filter过滤器执行的时候，它们执行的有限顺序是由它们在web.xml中从上到下配置的顺序决定的！！！

多个Filter过滤器执行的特点：

  1. 所有filter和目标资源默认都执行在同一个线程中!
  2. 多个Filter共同执行的时候，它们都是用同一个Request对象
